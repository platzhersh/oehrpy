# Building Open CIS — Article Series

oehrpy is part of the **Open CIS** project, documented in an ongoing Medium article series. These articles provide background, motivation, and technical context for the project.

## Articles

### Part 4: The openEHR SDK Landscape

**[Read on Medium](https://medium.com/@platzh1rsch/building-open-cis-part-4-the-openehr-sdk-landscape-1b93411ec279)**

A survey of existing openEHR SDKs across languages — Java, .NET, TypeScript, Python — and the gap that motivated building oehrpy. Covers:

- The existing SDK options and their maturity
- Why Python was underserved in the openEHR ecosystem
- The decision to build a new SDK from scratch
- Evaluation of three approaches: collaboration, archie-based, and BMM-to-Python generation

### Part 5: oehrpy — A Python SDK for openEHR

**[Read on Medium](https://medium.com/@platzh1rsch/building-open-cis-part-5-oehrpy-a-python-sdk-for-openehr-c9c90f46d075)**

The announcement and technical walkthrough of oehrpy. Covers:

- The SDK architecture and component design
- How RM classes are generated from openEHR specifications
- Template parsing and builder generation from OPT files
- FLAT format support and EHRBase integration
- Usage examples and getting started
