# Brand Guidelines

The Open CIS / oehrpy brand identity is built around **"The Archetype Brick"** — isometric interlocking bricks representing openEHR archetypes as composable building blocks.

## Brand Concept

Two stacked isometric bricks:

- **Top brick** (Archetype Blue) — Represents the archetype/template layer
- **Bottom brick** (Foundation Orange) — Represents the stable RM foundation

The bricks interlock via studs, symbolizing the composable, type-safe nature of the SDK.

## Color Palette

### Primary Colors

| Color | Hex | RGB | Usage |
|-------|-----|-----|-------|
| **Archetype Blue** | `#005EB8` | 0, 94, 184 | Primary brand color, top brick, "cis" in wordmark, links |
| **Foundation Orange** | `#F39200` | 243, 146, 0 | Secondary, bottom brick, accents, hover states |

### Neutrals

**Clinical Neutrals** — Slate scale (slate-900 through slate-50) for UI scaffolding, typography, and data-dense views.

## Typography

**Primary font:** [Inter](https://rsms.me/inter/)

- **Logo:** `open` (Inter Light) + `cis` (Inter Black, Archetype Blue)
- **Headings:** Inter Bold / Semibold
- **Body:** Inter Regular
- **Code:** JetBrains Mono

The weight contrast (Light vs Black) in the wordmark represents the transition from generic open-source foundations to robust clinical-specific tools.

## Logo Variants

| Variant | Usage |
|---------|-------|
| **Light mode lockup** | Brick icon + "open" (dark) + "cis" (blue) — for light backgrounds |
| **Dark mode lockup** | Brick icon + "open" (white) + "cis" (blue) — for dark backgrounds |
| **Social icon** | White brick on blue background — for avatars, social cards |
| **Glyph only** | "OC" in Inter Black, Archetype Blue — for tight spaces |
| **Favicon** | Simplified two-brick icon without studs — for browser tabs |
